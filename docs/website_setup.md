# 官网与下载页配置

本项目自带一个轻量官网首页和下载跳转入口，适合用于桌面软件发布、会员说明和商业化运营。

## 页面入口

- 官网首页：`/`
- 用户工作台：`/user`
- 下载跳转：`/download`
- 管理后台：`/admin`

`/download` 会读取后台配置的下载地址。如果没有配置下载地址，会回退到 `/user`。

## 推荐配置流程

1. 完成本机或服务端初始化。
2. 创建管理员账号。
3. 登录 `/admin`。
4. 打开“站点管理”。
5. 配置品牌、下载、协议、联系方式和公告。
6. 保存后访问 `/` 和 `/download` 验证展示与跳转。

## 站点配置项

Admin 后台的站点管理主要维护：

- 站点名称。
- 首页标题、关键词和描述。
- 官网地址。
- 下载地址。
- Logo 地址。
- 工作台标题和副标题。
- 登录页标题和副标题。
- 管理后台标题。
- 用户协议和隐私协议。
- 联系方式。
- 官方公告。

## 下载地址建议

安装包不要提交到 Git。推荐把安装包发布到：

- GitHub Releases。
- 自己的对象存储或 CDN。
- 自己的下载服务器。

然后把最终下载 URL 填到 Admin 后台的“下载地址”。这样官网首页和 `/download` 会统一跳转到最新安装包。

## 二开建议

- 官网模板：`app/templates/user/landing.html`
- 官网样式：`app/static/css/landing.css`
- 下载跳转：`app/views/user.py`
- 站点配置接口：`app/views/api.py`
- Admin 配置入口：`app/templates/user/admin.html`

如果要替换品牌、文案或布局，优先调整模板和样式；如果要增加新的站点配置项，再同步调整 Admin 表单、接口和配置读取逻辑。
